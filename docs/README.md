# 클로드 효율적 이용 가이드

> Claude를 더 효율적으로 사용하여 **토큰 30-50% 절감**, **캐시 히트율 70% 달성**, **일관된 고품질 결과물** 생성

## 📖 가이드 구성

이 가이드는 8개 섹션으로 구성되어 있으며, 각 섹션은 독립적으로 읽을 수 있습니다.

### [1. 시작하기 전에](sections/01-getting-started.md)
- **읽기 경로**: 초보자 / 고급 사용자 / 팀 리더별 가이드
- **용어 정의**: 토큰, 컨텍스트, 캐싱, 세션, 캐시 히트율
- **빠른 시작**: 5분 안에 적용 가능한 핵심 3가지
- **예상 시간**: 15-20분

### [2. 토큰 최적화 전략](sections/02-token-optimization.md)
- **11가지 기본 원칙**: 파일 재읽기 방지, 병렬 도구 호출, 컨텍스트 압축 등
- **캐싱 이해하기**: 프롬프트 캐싱 메커니즘 상세 설명
- **캐시 최적화**: 70% 이상 히트율 달성 전략
- **예상 시간**: 45-60분
- **🔥 가장 중요한 섹션**

### [3. 프로젝트 설정](sections/03-project-setup.md)
- **CLAUDE.md 작성**: 프로젝트 규칙 및 컨벤션 정의
- **.claudeignore 설정**: 불필요한 파일 제외 패턴
- **Hooks 설정**: 자동화 및 검증 훅
- **예상 시간**: 30-40분

### [4. 프롬프팅 베스트 프랙티스](sections/04-prompting.md)
- **공통 원칙**: 명확성, 컨텍스트, 구조화
- **코딩 작업**: 구현, 리팩토링, 디버깅 프롬프트
- **문서 작성**: README, API 문서, 기술 블로그
- **리뷰 작업**: 코드 리뷰, 아키텍처 리뷰
- **예상 시간**: 35-45분

### [5. 워크플로우 패턴](sections/05-workflows.md)
- **개인 워크플로우**: 일일 루틴, 대형 작업 관리, 컨텍스트 압축
- **팀 협업**: CLAUDE.md 표준, PR 리뷰, 템플릿 공유
- **예상 시간**: 25-30분

### [6. 품질 관리](sections/06-quality.md)
- **결과물 검증**: 코드 품질, 문서 품질 체크리스트
- **일반적인 실수**: 11가지 흔한 실수 패턴과 해결책
- **품질 보장**: 작업 전/중/후 체크리스트
- **예상 시간**: 30-35분

### [7. 측정 및 개선](sections/07-measurement.md)
- **추적할 메트릭**: 토큰 사용량, 캐시 히트율, 세션 효율성
- **캐시 분석**: 히트율 진단 및 개선 방법
- **최적화 체크리스트**: 자동/수동 점검 항목
- **예상 시간**: 25-30분

### [8. 고급 테크닉](sections/08-advanced.md)
- **Subagent 활용**: 작업 분산 및 병렬 처리
- **컨텍스트 관리**: 생명주기 관리, 압축 기법
- **메모리 시스템**: CLAUDE.md 최적화, 세션 간 정보 전달
- **캐시 친화적 구조**: 파일 구조, 타입 분리, 의존성 관리
- **예상 시간**: 40-45분

## 🛠️ 커맨드 도구

프로젝트 효율성을 자동으로 검사하는 커맨드:

```bash

/efficiency-audit-tokens   # 토큰 최적화만 (21개)
/efficiency-audit-setup    # 프로젝트 설정만 (10개)
/efficiency-audit-cache    # 캐시 친화성만 (15개)
```

자세한 내용은 `.claude/commands/` 참조.
