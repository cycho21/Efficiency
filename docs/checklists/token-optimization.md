# 토큰 최적화 체크리스트

이 체크리스트는 프로젝트의 Claude 사용 최적화 상태를 평가하고 개선점을 파악하는 도구입니다.

**목표 점수**: 80점 이상 (100점 만점)  
**권장 점검 주기**: 주 1회 또는 주요 변경 후

---

## 점수 계산 방법

각 항목은 pass/fail로 평가합니다:
- ✅ Pass: 해당 항목의 배점 획득
- ❌ Fail: 0점

**카테고리별 배점**:
- 프로젝트 설정: 30점 (항목당 5점 × 6개)
- 세션 관리: 20점 (항목당 5점 × 4개)
- 캐싱 최적화: 25점 (히트율 7점, 나머지 항목당 6점 × 3개)
- 파일 관리: 15점 (항목당 5점 × 3개)
- 프롬프팅: 10점 (항목당 2.5점 × 4개)

**총점 = 100점**

**평가 기준**:
- 90-100점: 우수 - 최적화가 매우 잘 되어 있음
- 80-89점: 양호 - 목표 달성, 일부 개선 여지 있음
- 70-79점: 보통 - 기본은 갖춤, 개선 필요
- 60-69점: 미흡 - 주요 항목 개선 시급
- 60점 미만: 불량 - 전면적인 최적화 필요

**실패시 조치**:

## 3. 캐싱 최적화 (25점)

### 3.1 캐시 히트율 70% 이상 (7점)
- [ ] 최근 세션들의 평균 캐시 히트율이 70% 이상이다

**확인 방법**:

**Claude API 사용시**:
```python
# API 응답에서 확인
cache_hit_rate = cache_read_input_tokens / input_tokens * 100
```

**Claude Code CLI 사용시**:
수동 추정: 반복되는 CLAUDE.md, 설정 파일 등의 토큰을 계산하여 추정합니다.

**계산 예시**:
```
전체 입력 토큰: 10,000
캐시에서 읽은 토큰: 7,000
히트율 = 7,000 / 10,000 = 70% ✅
```

**실패시 조치**:
- CLAUDE.md 수정 빈도 줄이기
- 세션을 5분 이내 간격으로 유지
- 자주 참조하는 파일을 세션 초기에 로드
- [메인 가이드 2.3절](../sections/02-token-optimization.md#23-캐시-히트율-최적화-전략) 참조

### 3.2 CLAUDE.md 안정성 (6점)
- [ ] CLAUDE.md 수정 빈도가 주 1회 미만이다

**확인 방법**:

```bash
git log --oneline .claude/CLAUDE.md | head -20
```

**이유**:
CLAUDE.md는 모든 세션에서 캐싱되므로 자주 수정하면 캐시가 무효화되어 비용이 증가합니다.

- 변경 전 여러 수정사항을 모아서 한 번에 적용
- 실험적인 내용은 임시 프롬프트로 제공

### 3.3 설정 파일 분리 (6점)
- [ ] 자주 참조하지만 변경되지 않는 설정 파일이 별도로 관리되고 있다

**예시**:
- API 스키마 정의
- 타입 정의 파일
- 공통 유틸리티 함수
- 코딩 스타일 가이드

**자가 평가 기준**:
- ✅ 핵심 설정을 모듈화하여 분리
- ✅ 변경 빈도가 낮은 파일 식별
- ✅ 세션마다 반복 참조되는 파일 최소화

**실패시 조치**:
자주 참조되는 파일 목록을 만들고, 변경되지 않는 파일을 별도로 관리하여 캐싱 효율을 높이세요.

---

### 3.4 세션 시작 최적화 (6점)
- [ ] 세션 시작시 공통 파일을 먼저 로드하는 패턴을 따른다

**권장 패턴**:
```
1. 세션 시작
```

**이점**:

**실패시 조치**:
- `session-start` Hook에 공통 파일 로드 로직 추가
- 작업 시작 전 체크리스트로 필요 파일 확인
- [메인 가이드 8.4절](../sections/08-advanced.md#84-캐시-친화적-프로젝트-구조) 참조

## 4. 파일 관리 (15점)

### 4.1 대용량 파일 제한 (5점)
- [ ] 500줄 이상의 파일이 5개 미만이다

**확인 방법**:

**실패시 조치**:

- 관심사 분리 (Separation of Concerns)
- 파일당 단일 책임 원칙 적용
- 필요시 특정 함수/클래스만 읽기

---

### 4.2 대용량 파일 분할 계획 (5점)

**정당한 사유 예시**:
- 자동 생성 파일 (수동 편집 X)
- 설정 파일 (JSON, YAML 등)
- 레거시 코드 (리팩토링 예정)

**실패시 조치**:
- 대용량 파일 목록 작성
- 각 파일별 분할 계획 수립
- 우선순위 지정 및 점진적 개선
- 분할이 불가능한 경우 사유 문서화

---

### 4.3 .claudeignore 적극 활용 (5점)
- [ ] 세션 중 불필요한 파일을 발견하면 즉시 .claudeignore에 추가하는 습관이 있다

**자가 평가 기준**:
- ✅ 빌드 결과물 발견시 즉시 제외
- ✅ 로그 파일, 임시 파일 패턴 추가
- ✅ 사용하지 않는 라이브러리 코드 제외
- ✅ .claudeignore를 정기적으로 검토 및 업데이트

**실패시 조치**:
- 세션 중 Claude가 불필요한 파일을 읽으면 즉시 .claudeignore 업데이트

### 5.4 템플릿 활용 (2.5점)
- [ ] 반복되는 작업에 대해 재사용 가능한 프롬프트 템플릿을 사용한다

**예시**:

템플릿 생성:
```markdown
## 코드 리뷰 템플릿

파일: {{FILE_PATH}}
리뷰 범위: {{SCOPE}} (보안/성능/스타일 중 선택)
우선순위: {{PRIORITY}}

다음 형식으로 리뷰해줘:
- Critical Issues: []
- Suggestions: []
- Good Practices: []
```

사용시:
```
{{FILE_PATH}} = "src/auth.service.ts"
{{SCOPE}} = "보안"
{{PRIORITY}} = "high"
```

**실패시 조치**:
- 자주 하는 작업 패턴 식별
- 각 패턴별 템플릿 작성
- 팀과 공유
- [프롬프트 템플릿 예시](../examples/prompt-templates.md) 참조

### 점수별 권장 조치

#### 90-100점: 우수
- 현재 상태 유지

- 목표 달성, 축하합니다!

- 주간 점검으로 개선 추적
- 캐시 히트율 모니터링 강화

- 우선순위 높은 항목부터 개선
  1. .claudeignore 설정

  3. 파일 재읽기 방지

- 1주 내 70점 이상 목표

#### 60점 미만: 불량
- 즉시 조치 필요
  1. [빠른 시작 가이드](../sections/01-getting-started.md#13-빠른-시작-가이드) 완료 (5분)

  4. 3일 내 재점검
